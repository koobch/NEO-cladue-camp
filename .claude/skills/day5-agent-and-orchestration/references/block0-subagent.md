# Block 0: Subagent 개념

> 공식 문서: https://docs.anthropic.com/en/docs/claude-code/sub-agents

## EXPLAIN

### 1. Subagent란?

**비유: "팀장이 팀원에게 일을 시키는 것"**

지금까지 Claude Code를 쓸 때, 하나의 Claude가 모든 일을 혼자 했습니다. 파일도 읽고, 분석도 하고, 결과도 정리하고. 마치 혼자서 모든 업무를 처리하는 1인 사업자 같았죠.

**Subagent**는 Claude가 **다른 Claude를 불러서 일을 시키는 것**입니다.

```
일반적인 사용:
  사용자 → Claude (혼자 다 함)

Subagent 사용:
  사용자 → Claude(팀장) → Claude(팀원 A): "이거 조사해와"
                        → Claude(팀원 B): "저거 분석해와"
                        → 결과를 모아서 보고
```

회사에서 팀장이 모든 일을 직접 하지 않고, 팀원에게 "이거 조사해서 보고해줘"라고 시키는 것과 같습니다. 팀원이 조사해서 결과를 가져오면, 팀장이 그걸 종합해서 판단합니다.

---

### 2. Subagent의 종류

Claude Code에서 사용하는 Subagent는 크게 3가지입니다:

| 종류 | 하는 일 | 비유 |
|------|---------|------|
| **Explore** | 읽기 전용 탐색. 파일을 읽고 구조를 파악 | "서류를 훑어보고 보고하는 인턴" |
| **General-purpose** | 범용 작업. 파일 수정, 분석, 코드 작성 등 | "뭐든 시키면 하는 실무자" |
| **특수 목적** | 코드 리뷰, 문서 업데이트 등 특정 역할 | "전문 분야가 있는 팀원" |

가장 자주 쓰는 건 **Explore**(탐색)와 **General-purpose**(범용)입니다.

---

### 3. 왜 Subagent가 필요한가?

혼자 다 하면 되는데 왜 굳이 Subagent를 쓸까요?

**이유 1: 복잡한 작업의 분업**

CSV 파일 3개를 동시에 분석해야 한다고 합시다. 혼자 하면 하나씩 순서대로 해야 합니다. Subagent를 쓰면 3개를 동시에 분석할 수 있습니다.

**이유 2: 컨텍스트 보호**

Claude는 대화가 길어지면 앞의 내용을 잊을 수 있습니다. Subagent는 각자 독립된 "메모장"을 가지고 있어서, 메인 Claude의 기억을 오염시키지 않습니다.

**이유 3: 병렬 처리**

```
순차 처리 (혼자): 파일A 분석(5분) → 파일B 분석(5분) → 파일C 분석(5분) = 15분
병렬 처리 (Subagent 3개): 파일A + 파일B + 파일C 동시 분석 = 5분
```

---

### 4. 이전 실습에서 이미 본 것

이전 실습에서 Claude가 파일을 탐색할 때 Explore 에이전트를 사용한 적이 있습니다. 그것이 바로 Subagent입니다.

오늘은 이것을 **의식적으로, 직접 지시해서** 사용해봅니다.

---

## EXECUTE

아래 순서대로 따라해보세요.

### Step 1. Explore 에이전트로 프로젝트 탐색

Claude Code에서 아래와 같이 입력하세요:

```
Explore 에이전트를 사용해서 이 프로젝트의 전체 구조를 파악해줘.
어떤 폴더와 파일이 있는지, 프로젝트의 목적이 뭔지 분석해서 보고해줘.
```

> **관찰 포인트**: Claude가 "서브에이전트를 시작합니다" 또는 "에이전트를 실행합니다"라는 메시지를 보여주는지 확인하세요. 이것이 Subagent가 동작하는 증거입니다.

### Step 2. Subagent로 병렬 분석

프로젝트에 CSV 파일이 있다면 아래를 입력하세요:

```
서브에이전트를 사용해서 data/ 폴더의 CSV 파일들을 동시에 분석하고, 각각의 요약을 만들어줘.
```

CSV 파일이 없다면 아래를 입력하세요:

```
서브에이전트를 사용해서 .claude/skills/ 폴더 안의 스킬 3개를 동시에 분석하고, 각각의 목적과 구조를 요약해줘.
```

> **관찰 포인트**: 여러 분석이 동시에 진행되는 것을 확인하세요. 하나씩 순서대로 하는 것보다 빠릅니다.

### Step 3. 차이 느끼기

Claude에게 아래와 같이 물어보세요:

```
방금 서브에이전트를 쓴 것과 혼자 분석하는 것의 차이가 뭐야?
```
